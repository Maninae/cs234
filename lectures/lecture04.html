Model-Free Control

From policy evaluation to control: SARSA, Q-learning, and scaling with function approximation and DQN.

Recap: Model-Free Policy Evaluation

In the previous lecture we developed methods for estimating the value function $V^\pi(s)$ of a fixed policy $\pi$ without access to the MDP's dynamics or reward model. We covered three main approaches.

Monte Carlo (MC) policy evaluation waits until the end of a complete episode, computes the empirical return $G_t$, and updates the value estimate:

$$V^\pi(s) \leftarrow (1 - \alpha) V^\pi(s) + \alpha\, G_t$$

Temporal-difference TD(0) updates after every single transition $(s, a, r, s')$, bootstrapping off the current value estimate:

$$V^\pi(s) \leftarrow V^\pi(s) + \alpha \bigl[ r + \gamma V^\pi(s') - V^\pi(s) \bigr]$$

Certainty equivalence maintains maximum-likelihood estimates of the dynamics and reward models from observed tuples, then solves for $V^\pi$ using standard dynamic-programming techniques. This approach exploits the Markov structure directly.

Batch MC and TD Convergence

When we use data repeatedly (batch/offline setting), MC and TD converge to different solutions. Given a finite set of $K$ episodes, repeatedly sampling an episode and applying updates, MC converges to the value that minimizes mean-squared error against observed returns, while TD(0) converges to the certainty-equivalence solution—the value function that would be correct if the maximum-likelihood MDP model were the true model.

Example — The AB Problem (Sutton & Barto, Ex. 6.4)

Consider two states $A$ and $B$ with $\gamma = 1$ and 8 episodes of experience: one episode $A, 0, B, 0$; six episodes of $B, 1$; and one episode $B, 0$. Both MC and TD agree that $V(B) = 0.75$. However, they disagree on $V(A)$. MC sets $V(A) = 0$ (the only observed return from $A$ was 0), whereas TD sets $V(A) = 0.75$ because the MLE model says $A$ always transitions to $B$, and $V(B) = 0.75$. TD exploits the Markov structure; MC does not.

Key metrics for comparing policy evaluation algorithms include data efficiency (how many samples are needed), computational efficiency (cost per update—both MC and TD(0) are $O(L)$ per episode of length $L$), and accuracy (bias, variance, and mean-squared error). TD exploits the Markov property and can be more data-efficient when the domain is truly Markov, but it introduces bias through bootstrapping.

From Evaluation to Control: Generalized Policy Iteration

Policy evaluation tells us how good a policy is; control tells us how to find a better one. The general recipe, called Generalized Policy Iteration (GPI), alternates between two steps:

  1. Policy evaluation: estimate $Q^\pi(s, a)$ (we use $Q$ rather than $V$ because, without a model, we cannot do the one-step lookahead required to extract a policy from $V$ alone).
  2. Policy improvement: set the new policy $\pi'$ to be greedy (or approximately greedy) with respect to the estimated $Q^\pi$.

A subtlety arises immediately: if $\pi$ is deterministic, we never take actions other than $\pi(s)$ in state $s$, so we cannot estimate $Q(s, a)$ for $a \neq \pi(s)$. This is the exploration problem—we must try different actions to learn about them, even though doing so might sacrifice short-term reward.

Epsilon-Greedy Policies

A simple and effective exploration strategy is the $\epsilon$-greedy policy. Given action-value estimates $Q(s, a)$ and $|A|$ available actions, we act as follows:

Definition — Epsilon-Greedy Policy

An $\epsilon$-greedy policy with respect to $Q(s,a)$ selects the greedy action with high probability and explores uniformly otherwise:

$$\pi(a \mid s) = \begin{cases} 1 - \epsilon + \frac{\epsilon}{|A|} & \text{if } a = \arg\max_{a'} Q(s, a') \\ \frac{\epsilon}{|A|} & \text{otherwise} \end{cases}$$

In words: with probability $1 - \epsilon$ take the best-known action; with probability $\epsilon$ choose uniformly at random among all actions (including the greedy one).

The crucial property of $\epsilon$-greedy policies is that policy improvement still works. If we form the $\epsilon$-greedy policy $\pi_{i+1}$ with respect to $Q^{\pi_i}$, the resulting policy is at least as good as $\pi_i$ everywhere.

Theorem — Monotonic Epsilon-Greedy Improvement

For any $\epsilon$-greedy policy $\pi_i$, the $\epsilon$-greedy policy $\pi_{i+1}$ with respect to $Q^{\pi_i}$ satisfies $V^{\pi_{i+1}}(s) \geq V^{\pi_i}(s)$ for all $s$. The proof follows from expanding:

$$Q^{\pi_i}(s, \pi_{i+1}(s)) = \frac{\epsilon}{|A|} \sum_{a \in A} Q^{\pi_i}(s,a) + (1-\epsilon)\max_a Q^{\pi_i}(s,a) \geq V^{\pi_i}(s)$$

The inequality holds because the max is at least as large as the average, ensuring $\pi_{i+1}$ is a monotonic improvement.

Monte Carlo Control

Combining MC evaluation with $\epsilon$-greedy improvement gives us a complete model-free control algorithm. After each episode, we update $Q$ estimates using observed returns, then recompute the $\epsilon$-greedy policy.

Algorithm: Monte Carlo On-Policy Control
  1. Initialize $Q(s,a) = 0$, $N(s,a) = 0$ for all $(s,a)$. Set $\epsilon = 1$, $k = 1$.
  2. Set $\pi_k = \epsilon\text{-greedy}(Q)$.
  3. Loop:
    1. Sample episode $k$: $(s_{k,1}, a_{k,1}, r_{k,1}, s_{k,2}, \ldots, s_{k,T})$ following $\pi_k$.
    2. Compute returns: $G_{k,t} = r_{k,t} + \gamma\, r_{k,t+1} + \gamma^2\, r_{k,t+2} + \cdots$
    3. For each first-visit $(s_t, a_t)$ in episode $k$:
      1. $N(s_t, a_t) \leftarrow N(s_t, a_t) + 1$
      2. $Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \frac{1}{N(s_t, a_t)} \bigl[ G_{k,t} - Q(s_t, a_t) \bigr]$
    4. $k \leftarrow k + 1$, $\epsilon \leftarrow 1/k$.
    5. $\pi_k \leftarrow \epsilon\text{-greedy}(Q)$. // Policy improvement

GLIE: Greedy in the Limit of Infinite Exploration

For MC control to converge to the optimal action-value function, we need the exploration schedule to satisfy the GLIE condition:

Definition — GLIE

A learning policy satisfies Greedy in the Limit of Infinite Exploration (GLIE) if:

  1. All state-action pairs are visited infinitely often: $\lim_{i \to \infty} N_i(s,a) = \infty$ for all $(s, a)$.
  2. The behavior policy converges to the greedy policy as $i \to \infty$.

A simple GLIE strategy is $\epsilon$-greedy with $\epsilon_i = 1/i$, which decays to zero while ensuring sufficient exploration in early episodes.

Theorem — GLIE MC Control Convergence

GLIE Monte Carlo control converges to the optimal state-action value function: $Q(s,a) \to Q^*(s,a)$ for all $(s, a)$.

On-Policy vs. Off-Policy Learning

Before introducing TD-based control algorithms, it is worth clarifying a fundamental distinction in how agents use experience.

Insight — On-Policy vs. Off-Policy

On-policy methods learn the value of the policy currently being followed. The same policy that generates experience is the one being evaluated and improved. SARSA is the canonical example.

Off-policy methods learn the value of a target policy $\pi$ while following a different behavior policy $\pi_b$. This decouples exploration (which $\pi_b$ handles) from the policy being optimized. Q-learning is the canonical example—it estimates $Q^*$ regardless of the behavior policy, as long as the behavior policy provides sufficient exploration.

Off-policy learning is more flexible (it can reuse data, learn from demonstrations, or learn about multiple policies simultaneously), but it introduces new challenges around sample corrections and stability.

SARSA: On-Policy TD Control

SARSA applies temporal-difference ideas to the control problem. Instead of waiting for full episode returns as in MC, it updates $Q$ after every transition. The name comes from the quintuple of values used in each update: $(S_t, A_t, R_t, S_{t+1}, A_{t+1})$.

Definition — SARSA Update Rule

After observing transition $(s_t, a_t, r_t, s_{t+1})$ and choosing the next action $a_{t+1} \sim \pi(s_{t+1})$, update:

$$Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha \bigl[ r_t + \gamma\, Q(s_{t+1}, a_{t+1}) - Q(s_t, a_t) \bigr]$$

The term in brackets is the TD error: the difference between the bootstrapped target $r_t + \gamma\, Q(s_{t+1}, a_{t+1})$ and the current estimate $Q(s_t, a_t)$. Because SARSA uses the action $a_{t+1}$ actually taken under the current policy, it is an on-policy algorithm.

The full algorithm interleaves this TD update with $\epsilon$-greedy policy improvement at every step:

Algorithm: SARSA (On-Policy TD Control)
  1. Initialize $Q(s,a)$ for all $s \in \mathcal{S}, a \in \mathcal{A}$. Set $t = 0$, initial state $s_t = s_0$.
  2. Choose $a_t \sim \pi(s_t)$ where $\pi$ is $\epsilon$-greedy w.r.t. $Q$.
  3. Loop:
    1. Take action $a_t$, observe $(r_t, s_{t+1})$.
    2. Choose $a_{t+1} \sim \pi(s_{t+1})$. // next action from current policy
    3. $Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha \bigl[ r_t + \gamma\, Q(s_{t+1}, a_{t+1}) - Q(s_t, a_t) \bigr]$
    4. $\pi(s_t) \leftarrow \epsilon\text{-greedy}(Q)$. // improve policy
    5. $t \leftarrow t + 1$.

Q-Learning: Off-Policy TD Control

Q-learning modifies the SARSA update in a subtle but consequential way: instead of using the action $a_{t+1}$ that was actually taken, it bootstraps off the best possible action at the next state. This single change makes Q-learning an off-policy algorithm that directly estimates $Q^*$ regardless of the behavior policy.

Definition — Q-Learning Update Rule

After observing $(s_t, a_t, r_t, s_{t+1})$, update:

$$Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha \bigl[ r_t + \gamma \max_{a'} Q(s_{t+1}, a') - Q(s_t, a_t) \bigr]$$

The key difference from SARSA is the $\max_{a'}$ in the target. Q-learning asks: "What is the best I could do from $s_{t+1}$?" regardless of what action the behavior policy will actually take. This makes Q-learning a direct approximation of the Bellman optimality equation.

Algorithm: Q-Learning with Epsilon-Greedy Exploration
  1. Initialize $Q(s,a)$ for all $s \in \mathcal{S}, a \in \mathcal{A}$. Set $t = 0$, initial state $s_t = s_0$.
  2. Set behavior policy $\pi_b = \epsilon\text{-greedy}(Q)$.
  3. Loop:
    1. Take $a_t \sim \pi_b(s_t)$. // sample from behavior policy
    2. Observe $(r_t, s_{t+1})$.
    3. $Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha \bigl[ r_t + \gamma \max_a Q(s_{t+1}, a) - Q(s_t, a_t) \bigr]$
    4. $\pi_b(s_t) \leftarrow \epsilon\text{-greedy}(Q)$.
    5. $t \leftarrow t + 1$.
Example — SARSA vs. Q-Learning on Mars Rover

Consider a Mars rover with 7 states, two actions $a_1, a_2$, rewards $r(\cdot, a_1) = [1, 0, 0, 0, 0, 0, 10]$ and $r(\cdot, a_2) = [0, 0, 0, 0, 0, 0, 5]$, $\gamma = 1$, and $\alpha = 0.5$. Suppose the rover is in state $s_6$, takes $a_1$, receives reward $0$, transitions to $s_7$, then takes $a_2$ receiving reward $5$.

With SARSA, the update uses the actual next action $a_2$: $Q(s_6, a_1) = 0.5 \cdot 0 + 0.5 \cdot (0 + Q(s_7, a_2)) = 0.5 \cdot 5 = 2.5$.

With Q-learning, the update uses the max over actions: $Q(s_6, a_1) = 0 + 0.5 \cdot (0 + \max_{a'} Q(s_7, a') - 0) = 0.5 \cdot 10 = 5$.

Q-learning gives a higher value because it "assumes" the best action will be taken next, whereas SARSA accounts for the fact that the $\epsilon$-greedy policy sometimes explores suboptimal actions.

Convergence of Q-Learning

Theorem — Q-Learning Convergence

Q-learning for finite-state, finite-action MDPs converges to the optimal action-value function, $Q(s,a) \to Q^*(s,a)$, under two conditions:

  1. The behavior policy sequence $\pi_t(a \mid s)$ satisfies the GLIE condition (all state-action pairs visited infinitely often).
  2. The step sizes satisfy the Robbins-Monro conditions: $$\sum_{t=1}^{\infty} \alpha_t = \infty \qquad \text{and} \qquad \sum_{t=1}^{\infty} \alpha_t^2 < \infty$$

For example, $\alpha_t = 1/t$ satisfies these requirements. The convergence proof relies on stochastic approximation theory, the contraction property of the Bellman optimality operator, and bounded rewards.

Scaling Up: Value Function Approximation

Tabular methods maintain a separate value for every state-action pair, which is infeasible when the state space is large or continuous (think of pixel-based observations in Atari, or joint angles of a robot). Value function approximation (VFA) replaces the lookup table with a parameterized function $\hat{Q}(s, a; \mathbf{w}) \approx Q^\pi(s, a)$, offering three key advantages:

  • Memory: a compact parameter vector $\mathbf{w}$ instead of $|\mathcal{S}| \times |\mathcal{A}|$ entries.
  • Computation: faster updates when the number of parameters is much smaller than the state-action space.
  • Generalization: updating $\hat{Q}$ at one state-action pair automatically changes estimates at similar states, reducing the experience needed to learn.

Learning with Stochastic Gradient Descent

Suppose an oracle could provide the true value $Q^\pi(s,a)$ at any query point. We would minimize the mean-squared error:

$$J(\mathbf{w}) = \mathbb{E}_\pi \bigl[ (Q^\pi(s,a) - \hat{Q}(s,a;\mathbf{w}))^2 \bigr]$$

Gradient descent gives the update direction:

$$\Delta \mathbf{w} = -\frac{1}{2}\alpha\, \nabla_\mathbf{w} J(\mathbf{w}) = \alpha\, \mathbb{E}_\pi \bigl[ (Q^\pi(s,a) - \hat{Q}(s,a;\mathbf{w})) \nabla_\mathbf{w} \hat{Q}(s,a;\mathbf{w}) \bigr]$$

In practice, we sample transitions and use stochastic gradient descent (SGD), which replaces the expectation with a single-sample estimate. The expected SGD update equals the full gradient, making it an unbiased estimator.

Monte Carlo VFA

Since the return $G_t$ is an unbiased (but noisy) sample of $Q^\pi(s_t, a_t)$, we substitute it for the true value. MC VFA reduces to supervised learning on state-action/return pairs $\langle (s_1, a_1), G_1 \rangle, \langle (s_2, a_2), G_2 \rangle, \ldots$ with the gradient update:

$$\Delta \mathbf{w} = \alpha \bigl( G_t - \hat{Q}(s_t, a_t; \mathbf{w}) \bigr) \nabla_\mathbf{w} \hat{Q}(s_t, a_t; \mathbf{w})$$

TD(0) VFA

For TD(0), the target $r + \gamma \hat{V}(s'; \mathbf{w})$ replaces the true value. This introduces three layers of approximation: sampling (single transition instead of expectation), bootstrapping (using $\hat{V}$ instead of true $V^\pi$), and function approximation (parameterized function instead of a table). The update becomes:

$$\Delta \mathbf{w} = \alpha \bigl( r + \gamma\, \hat{V}(s'; \mathbf{w}) - \hat{V}(s; \mathbf{w}) \bigr) \nabla_\mathbf{w} \hat{V}(s; \mathbf{w})$$

Note that the gradient is only taken with respect to $\hat{V}(s; \mathbf{w})$, not the target—this is a semi-gradient method. Despite the theoretical complications this introduces, semi-gradient TD works well in practice with linear function approximation.

Control with Function Approximation

To perform control with VFA, we combine approximate policy evaluation (using MC or TD targets with a parameterized $\hat{Q}$) with $\epsilon$-greedy policy improvement. The update rules for each method generalize naturally:

MC control:

$$\Delta \mathbf{w} = \alpha \bigl( G_t - \hat{Q}(s_t, a_t; \mathbf{w}) \bigr) \nabla_\mathbf{w} \hat{Q}(s_t, a_t; \mathbf{w})$$

SARSA with VFA:

$$\Delta \mathbf{w} = \alpha \bigl( r + \gamma\, \hat{Q}(s', a'; \mathbf{w}) - \hat{Q}(s, a; \mathbf{w}) \bigr) \nabla_\mathbf{w} \hat{Q}(s, a; \mathbf{w})$$

Q-learning with VFA:

$$\Delta \mathbf{w} = \alpha \bigl( r + \gamma \max_{a'} \hat{Q}(s', a'; \mathbf{w}) - \hat{Q}(s, a; \mathbf{w}) \bigr) \nabla_\mathbf{w} \hat{Q}(s, a; \mathbf{w})$$

However, combining function approximation with control introduces potential instability, particularly when certain conditions coincide.

The Deadly Triad

Insight — The Deadly Triad

The combination of three individually useful techniques can cause divergence or oscillation in value estimates:

  1. Function approximation — representing the value function with a parameterized model rather than a table.
  2. Bootstrapping — updating estimates based on other estimates (as in TD methods), rather than waiting for complete returns.
  3. Off-policy learning — learning about one policy while following another (as in Q-learning).

Any two of these three can be combined safely. The danger emerges when all three are present simultaneously. Intuitively, the Bellman backup is a contraction in the tabular case, but projecting the result back onto the function approximation space can be an expansion, causing updates to amplify rather than shrink errors. Off-policy learning exacerbates this because the distribution of states visited under the behavior policy may differ significantly from the distribution relevant to the target policy.

Deep Q-Networks (DQN)

The 2015 paper "Human-level control through deep reinforcement learning" by Mnih et al. demonstrated that Q-learning with a deep neural network could learn to play Atari games directly from pixels, matching or exceeding human performance in many games. This was a landmark result, but making it work required addressing the two main problems that arise when naively combining neural networks with Q-learning:

  • Correlated samples: consecutive transitions in an episode are highly correlated, which violates the i.i.d. assumption underlying SGD and can cause learning to diverge.
  • Non-stationary targets: since the target $r + \gamma \max_{a'} \hat{Q}(s', a'; \mathbf{w})$ depends on the same weights $\mathbf{w}$ being updated, the target shifts with every gradient step, creating a moving-target problem.

DQN addresses both issues with two key innovations: experience replay and fixed target networks.

Experience Replay

Instead of learning only from the most recent transition, DQN stores transitions $(s, a, r, s')$ in a large replay buffer $\mathcal{D}$. At each training step, a random minibatch is sampled from $\mathcal{D}$ and used for the gradient update. This breaks temporal correlations (since the minibatch contains transitions from many different episodes and time steps) and dramatically improves data efficiency by reusing each experience many times.

Fixed Q-Targets

To reduce the non-stationarity of the target, DQN maintains a separate target network with parameters $\mathbf{w}^-$ that are held fixed for $C$ update steps. The target becomes:

$$y_i = r_i + \gamma \max_{a'} \hat{Q}(s_{i+1}, a'; \mathbf{w}^-)$$

The main network parameters $\mathbf{w}$ are updated to minimize $(y_i - \hat{Q}(s_i, a_i; \mathbf{w}))^2$, while $\mathbf{w}^-$ is only periodically synchronized with $\mathbf{w}$ (every $C$ steps). This provides a stable regression target for many consecutive updates, significantly improving training stability. Note that while this doubles the memory requirements (two copies of the network weights), it does not double computation time since the target network is only used for forward passes, not gradient computation.

Algorithm: Deep Q-Network (DQN)
  1. Input: target update frequency $C$, learning rate $\alpha$.
  2. Initialize replay buffer $\mathcal{D} = \{\}$, network weights $\mathbf{w}$, target weights $\mathbf{w}^- = \mathbf{w}$, $t = 0$.
  3. Get initial state $s_0$.
  4. Loop:
    1. Choose $a_t$ using $\epsilon$-greedy policy w.r.t. $\hat{Q}(s_t, \cdot\,; \mathbf{w})$.
    2. Execute $a_t$, observe $(r_t, s_{t+1})$.
    3. Store $(s_t, a_t, r_t, s_{t+1})$ in $\mathcal{D}$.
    4. Sample random minibatch $\{(s_j, a_j, r_j, s_{j+1})\}$ from $\mathcal{D}$.
    5. For each $(s_j, a_j, r_j, s_{j+1})$ in minibatch:
      1. If $s_{j+1}$ is terminal: $y_j = r_j$
      2. Else: $y_j = r_j + \gamma \max_{a'} \hat{Q}(s_{j+1}, a'; \mathbf{w}^-)$
      3. $\Delta \mathbf{w} = \alpha (y_j - \hat{Q}(s_j, a_j; \mathbf{w})) \nabla_\mathbf{w} \hat{Q}(s_j, a_j; \mathbf{w})$
    6. $t \leftarrow t + 1$.
    7. If $t \bmod C = 0$: $\mathbf{w}^- \leftarrow \mathbf{w}$ // sync target network

DQN in Practice: Atari Results

In the Atari domain, the DQN architecture takes as input a stack of the last 4 raw game frames (pixels), processes them through convolutional layers, and outputs $Q(s, a)$ for each of the 18 possible joystick/button positions. The reward signal is simply the change in game score at each step. Remarkably, the same network architecture and hyperparameters were used across all games.

Example — Ablation Study: What Makes DQN Work?

Ablation experiments reveal that experience replay is by far the most important ingredient. Comparing average scores across several games:

  • Linear baseline: reasonable scores with simple features.
  • Deep network alone (no replay, no fixed targets): often worse than the linear baseline, demonstrating that simply adding depth does not help and can hurt.
  • DQN with fixed targets only: moderate improvement over the deep baseline.
  • DQN with replay only: large improvements (e.g., Breakout jumps from 3 to 241).
  • Full DQN (replay + fixed targets): best performance (Breakout reaches 317, River Raid reaches 7447).

Beyond decorrelating samples, replay also provides massive data efficiency gains—each transition is used for many gradient updates instead of just one.

Beyond DQN

The success of DQN sparked a wave of improvements to deep Q-learning:

  • Double DQN (Van Hasselt et al., 2016): addresses the overestimation bias of Q-learning by decoupling action selection from evaluation in the target.
  • Prioritized Experience Replay (Schaul et al., 2016): samples transitions with high TD error more frequently, focusing learning where it is most needed.
  • Dueling DQN (Wang et al., 2016): separates the network into a state-value stream and an advantage stream, improving learning when many actions have similar values.

Summary

This lecture covered the full pipeline from model-free evaluation to model-free control, and then scaled these ideas to large state spaces using function approximation.

  • Generalized Policy Iteration (GPI) alternates evaluation and improvement. Using $Q$ values (rather than $V$) avoids the need for a dynamics model.
  • Epsilon-greedy exploration ensures all actions are tried while mostly exploiting current knowledge. Under GLIE conditions ($\epsilon \to 0$ at rate $1/k$), MC control converges to $Q^*$.
  • SARSA is on-policy TD control: it updates $Q(s_t, a_t)$ toward $r_t + \gamma Q(s_{t+1}, a_{t+1})$ using the action actually taken.
  • Q-learning is off-policy TD control: it updates toward $r_t + \gamma \max_{a'} Q(s_{t+1}, a')$, directly estimating $Q^*$ regardless of the behavior policy.
  • Function approximation replaces the lookup table with $\hat{Q}(s, a; \mathbf{w})$, enabling generalization across states. SGD-based updates extend naturally from the tabular case.
  • The deadly triad—function approximation + bootstrapping + off-policy learning—can cause divergence. Care must be taken when all three are present.
  • DQN stabilizes deep Q-learning through experience replay (breaks sample correlations, improves data efficiency) and fixed target networks (stabilizes the regression target). These innovations enabled human-level Atari play from raw pixels.
